Quick Start Guide

This app lets you create and explore fractal images — infinitely detailed patterns built from a handful of simple rules. You don't need any maths background to get started: just add transformations, drag the handles in the preview, and watch the fractal evolve in real time.

The two ways to tweak a transformation

Each transformation appears as a small interactive square in the sidebar. You can change it in two ways:

  • Drag directly on the preview square — grab a corner or the centre to move, rotate, or resize that piece of the fractal. The result updates live on the canvas.
  • Type numbers in the matrix fields (m00 – m12) — for when you want exact, reproducible values. See the "How the maths works" section below if you'd like to understand what those numbers mean.

Common actions

  • Add — add another transformation to make the fractal more complex.
  • Shuffle — randomise all transformations and discover unexpected shapes.
  • Reset — you can always return to the initial state by selecting Sierpinski Triangle from the Import menu.
  • Save / Export — download your configuration as a JSON file, or save the canvas as a PNG image. Cool Tip: Saved PNGs have your fractal data hidden inside them — you can load a fractal back just by importing its PNG!

Drag handles cheat-sheet

  • Centre of square: move (translate) the piece
  • Top-left red dot: rotate
  • Top-right red square: scale uniformly (zoom in/out)
  • Blue midpoint circles: stretch on X or Y axis independently
  • Mouse wheel: zoom / scale
  • Ctrl + drag: rotate (alternative shortcut)

What are affine transformations?

Each affine transformation is a recipe for taking a point on the canvas and moving it somewhere new. The recipe involves six numbers — m00 through m12 — which control:

  • m00 and m11 — how much to scale (stretch or shrink) the shape
  • m01 and m10 — how much to rotate or shear (skew) the shape
  • m02 and m12 — how far to shift the shape horizontally and vertically

Mathematically, the new position of a point is calculated as: 

x' = m00·x + m01·y + m02
y' = m10·x + m11·y + m12

In matrix form (useful if you're familiar with linear algebra): 

[ x' ]   [ m00  m01  m02 ] [ x ]
[ y' ] = [ m10  m11  m12 ] [ y ]
[ 1  ]   [  0    0    1  ] [ 1 ]

Don't worry if the matrix notation looks unfamiliar — everything you can do with numbers you can also do visually using the drag handles.

How the fractal is built — the intuitive picture

Imagine you have a drawing. You apply each of your transformations to that drawing — shrinking it, rotating it, shifting it — and you paste all the tiny copies back together. That becomes your new drawing. Repeat this process over and over, and the result gradually converges into a fractal.

The classic example is the Sierpiński triangle: start with a filled triangle, apply three "shrink-and-shift" rules, and after just a few iterations the familiar triangle-within-triangle pattern appears.

Each step applies the same three rules to the previous result. After only a handful of iterations, the fractal is already clearly recognisable.

The remarkable thing is that no matter what shape you start with — a square, a photograph, a random scribble — the system always converges to the same final fractal. The rules alone determine the outcome.

About fractals

What is a fractal?

A fractal is a shape that looks similar to itself no matter how closely you zoom in. This property is called self-similarity. Fractals are not just mathematical curiosities — they appear everywhere in nature: coastlines, snowflakes, ferns, river networks, lightning bolts, and the branching of your lungs all display fractal-like structure.

Famous fractals to explore

  • Sierpiński Triangle — described by Polish mathematician Wacław Sierpiński (1882–1969) in 1915. The default preset in this app produces this shape.
  • Koch Snowflake — a curve with infinite perimeter enclosing a finite area. One of the earliest fractal examples.
  • Barnsley Fern — a fern leaf generated by exactly four affine transformations. A perfect demonstration of how complex natural-looking forms arise from simple IFS rules.
  • Mandelbrot Set — arguably the most famous fractal, generated by a different method (complex number iteration) but sharing the same spirit of infinite complexity from simple rules.
  • Julia Sets — close relatives of the Mandelbrot set, each with its own distinctive shape.

Iterated Function Systems (IFS)

This app uses a technique called an Iterated Function System. An IFS is simply a collection of transformations (like the ones you edit in the sidebar). When applied repeatedly, they always converge to the same fractal shape — called the attractor of the system.

This convergence is guaranteed by the Banach Fixed-Point Theorem: as long as each transformation shrinks distances (i.e. is contractive), the system is mathematically guaranteed to settle on a unique shape, regardless of the starting image.

IFS were popularised by mathematician Michael Barnsley in the 1980s, and have been used in applications ranging from image compression to procedural texture generation in video games.

Technical implementation

This is a pure client-side Angular application built to handle high-complexity fractals smoothly, even in the browser.

The compositional algorithm

The step-by-step approach (apply rules → get new image → repeat) is intuitive but slow at high resolutions, because each iteration redraws every pixel. Instead, this app uses a compositional approach: it pre-computes all possible combinations of transformations and draws the final shapes directly, skipping intermediate images entirely. Remark: The combinations are computed 'depth first' to avoid storing too many intermediate affine transformations in memory during the computation.

For 3 transformations at iteration depth 5, this means drawing 3⁵ = 243 tiny shapes directly onto the canvas — far more efficient than iterating step by step.

Performance techniques

  • Web Workers — heavy computation runs in a background thread, so the UI stays smooth at 60 fps while the fractal renders.
  • Asynchronous batching — results are sent to the canvas progressively in batches, so you see the fractal build up gradually rather than waiting for everything at once.
  • Backpressure management — if the rendering thread falls behind, the worker pauses automatically to avoid memory buildup.
  • Transferable objects — large data arrays (Float32Array) are passed between threads without copying, keeping transfers fast.

Metadata Embedding

To make sharing easier, this app uses custom PNG tEXt chunks. When you save an image, the entire fractal configuration is embedded into the file as metadata. This means the PNG is not just a picture, but also a portable data container that can be re-imported later to restore the exact same transformation set.
